# Документация по инструменту локализации для Unity

[cite_start]Этот инструмент представляет собой комплексное решение для локализации игр и приложений в Unity[cite: 60]. [cite_start]Он позволяет автоматизировать процесс сбора текста и ассетов, управлять переводами через мощный интерфейс, интегрироваться с сервисами машинного перевода и динамически обновлять локализованный контент в игре[cite: 61].

## 👑 Ключевые возможности

### Core Features & Workflow
* [cite_start]**Мощный редактор таблицы переводов**: Полноценный интерфейс для управления всеми ключами и переводами прямо в Unity с полной поддержкой Undo/Redo, автосохранением и восстановлением сессии[cite: 2].
* [cite_start]**Умная группировка ключей**: Автоматически объединяет ключи для множественного числа и рода (например, `apple_one`, `apple_few`) в сворачиваемые группы[cite: 3].
* [cite_start]**Продвинутая система парсинга**: Автоматический поиск текста в сценах, префабах, UI Toolkit (UXML) и C# скриптах (включая поля с атрибутом `[LocalizableField]`)[cite: 4, 63].
* [cite_start]**Локализация любых ассетов**: Поддержка спрайтов, текстур, аудиоклипов, материалов, видео, префабов и других типов ассетов[cite: 5, 64].
* [cite_start]**Автоматический перевод**: Интеграция с DeepL, Google Translate и Microsoft Translator с индивидуальной настройкой[cite: 6, 66].
* [cite_start]**Live Preview в редакторе**: Предпросмотр любого языка прямо в сцене без запуска игры[cite: 7, 68].
* [cite_start]**4 режима генерации ключей**: Гибкая настройка для разных этапов разработки[cite: 8].
* [cite_start]**Безопасная миграция ключей**: Автоматическое сохранение всех переводов при смене режима генерации ключей[cite: 9, 129].
* [cite_start]**Live Updates (Живые обновления)**: Загрузка актуальных переводов с удаленного URL (Google Sheets или JSON) при старте игры [cite: 10, 71] [cite_start]с кэшированием[cite: 11].
* [cite_start]**Поддержка Plural & Gender**: Корректная обработка форм множественного числа и рода[cite: 12, 69].
* [cite_start]**Полная поддержка UI Toolkit**: Глубокая интеграция с UXML[cite: 13, 107].
* [cite_start]**Менеджер бэкапов**: Встроенный инструмент для создания и восстановления резервных копий[cite: 14, 72].

### Automation & Tools
* [cite_start]**Контекстный поиск по проекту**: Асинхронный инструмент "Find" для мгновенного нахождения любого локализованного элемента[cite: 16, 17, 167].
* [cite_start]**Детальный отчет о парсинге**: На вкладке 'Report' доступен подробный отчет со всеми изменениями (Added, Updated, Removed и т.д.)[cite: 19, 21, 166].
* [cite_start]**Контекстное меню в инспекторе**: Быстрый доступ к анализу локализации прямо из инспектора компонента[cite: 22, 113].
* [cite_start]**Импорт и экспорт данных**: Поддержка форматов CSV, XML, YAML и XLIFF, а также прямой импорт из Google Sheets[cite: 23, 24, 67].
* [cite_start]**Автоматическое назначение компонентов**: Анализ всего проекта одной кнопкой для автоматического добавления скриптов локализации[cite: 25].
* [cite_start]**Гибкая настройка сканирования** и автоматическое создание папок для ассетов[cite: 28, 29].

### Technical & Quality of Life
* [cite_start]**Полная поддержка RTL (Right-to-Left)**: Корректное отображение языков с письмом справа налево[cite: 33, 70].
* [cite_start]**Продвинутый многострочный редактор**: Сравнение с исходным текстом и валидация плейсхолдеров в реальном времени[cite: 34].
* [cite_start]**Умная валидация плейсхолдеров**: Инструмент предупредит, если количество или имена плейсхолдеров (`{0}`, `{username}`) не совпадают с оригиналом[cite: 35].
* [cite_start]**Защита плейсхолдеров и переносов строк** от повреждения автоматическим переводчиком[cite: 37].
* [cite_start]**"Защита от дурака" (Preview Protector)**: Автоматически отменяет изменения от Live Preview перед сохранением сцены[cite: 38, 279].
* [cite_start]**Защита от сбоев редактора (Preview Crash Protector)**: Восстанавливает исходное состояние объектов при следующем запуске Unity[cite: 39, 122].
* [cite_start]**Безопасное хранение API ключа** локально на машине пользователя и в зашифрованном виде для билда[cite: 45, 46, 276].
* [cite_start]**Автоматический фолбэк** на исходный язык, если перевод отсутствует[cite: 44].

## 🚀 Быстрый старт и настройка

#### 1. Установка
Скопируйте ассет в папку вашего проекта. [cite_start]Инструмент автоматически предложит установить необходимые зависимости (Newtonsoft Json, Editor Coroutines) и опциональные (Arabic Support)[cite: 75, 76, 77, 78]. [cite_start]После установки откройте окно через **Tools -> Localization Tool**[cite: 80].

#### 2. Первоначальная настройка
1.  [cite_start]**Настройка языков**: На вкладке **Settings -> Language Management** выберите ваш основной язык как `Source Language` и включите (`Enabled`) все целевые языки[cite: 83, 84].
2.  [cite_start]**Указание контента**: На вкладке **Content** добавьте ваши сцены в `Scenes to Parse` и папки с префабами в `Prefab Folders`[cite: 85, 86].
3.  **Первый запуск**: Перейдите на вкладку **Actions** и нажмите **Update Keys**. [cite_start]Инструмент просканирует проект и автоматически добавит необходимые компоненты[cite: 87, 88].

## ⚙️ Основные компоненты

**Важно:** Все компоненты добавляются на объекты автоматически при нажатии `Update Keys`. [cite_start]Вам не нужно делать это вручную[cite: 90, 91].

* [cite_start]`LocalizedText`: Основной компонент для текста (Text, TMP_Text)[cite: 95].
* [cite_start]`LocalizedAsset`: Для подмены ассетов (Sprite, AudioClip, Material)[cite: 101]. [cite_start]Корректно работает с `Play on Awake`[cite: 102].
* [cite_start]`LocalizedPrefab`: Для неразрушающей локализации префабов[cite: 104, 42].
* [cite_start]`UITKLocalization`: Управляет локализацией внутри UI Toolkit (UXML) документов[cite: 107].
* [cite_start]`LocalizedBehaviour`: Позволяет вашим скриптам реагировать на смену языка через атрибут `[OnLanguageChange]`[cite: 111, 219].

## 🛠️ Окно "Localization Tool"

#### Вкладка "Settings" (Настройки)
Центр управления, где вы настраиваете глобальные правила.
* **Key Generation Mode**: Выберите способ создания ключей. [cite_start]`UseTextAsKey` для прототипов, `AutoGenerateKeysOnly` для продакшена[cite: 125, 126, 127]. [cite_start]Миграция между режимами безопасна[cite: 129].
* [cite_start]**Language Management**: Настройте список языков, шрифты для них и опцию RTL[cite: 130, 131].
* [cite_start]**Live Updates**: Настройки для загрузки переводов с удаленного сервера[cite: 138].

#### Вкладка "Content" (Контент)
Здесь вы указываете, где искать текст для локализации.
* [cite_start]**Scenes to Parse** и **Prefab Folders**: Списки сцен и папок для анализа[cite: 142, 143].
* [cite_start]**Dynamic Texts**: Строки, которые создаются только в коде (например, "Game Over")[cite: 144].
* [cite_start]**Parsing Ignores**: Укажите объекты или компоненты для игнорирования[cite: 145].

#### Вкладка "Actions" (Действия)
Главная рабочая вкладка.
* [cite_start]**Update Keys**: Запускает парсер[cite: 150].
* [cite_start]**Open Translation Editor**: Открывает мощный редактор переводов[cite: 151].
* [cite_start]**Data Management**: Импорт/Экспорт в CSV, XML, XLIFF и из Google Sheets[cite: 152].
* [cite_start]**Auto-Translation**: Заполняет пустые поля с помощью DeepL, Google или Microsoft[cite: 153].

#### Вкладка "Report" (Отчет)
Подробный отчет после каждого парсинга. [cite_start]Позволяет отследить все добавленные, удаленные, обновленные и дублирующиеся ключи[cite: 165, 166]. [cite_start]Кнопка `Find` поможет мгновенно найти любой объект в проекте[cite: 167].

Отдельное окно (`Tools -> Localization -> Translation Table Editor`) для ручного редактирования.
* [cite_start]**Умная группировка** ключей для множественного числа и рода[cite: 172].
* [cite_start]**Продвинутое редактирование** с окном `MultiLineEditWindow`, которое показывает исходный текст и валидирует плейсхолдеры в реальном времени[cite: 173, 174].
* [cite_start]**Полная поддержка Undo/Redo** (Ctrl+Z / Ctrl+Y)[cite: 179].
* [cite_start]**Встроенный менеджер бэкапов** и автосохранение сессии[cite: 180].

## 💡 Примеры использования

#### Атрибут `[LocalizableField]`
Предпочтительный способ для локализации полей в инспекторе. [cite_start]Работает со строками, списками и даже вложенными классами[cite: 191, 192].
```csharp
using Ankonoanko.Localization;
using UnityEngine;
using System.Collections.Generic;

public class QuestManager : MonoBehaviour
{
    // Парсер найдет это поле
    [LocalizableField]
    private string defaultQuestFailedMessage = "You have failed the quest.";

    // Можно задать собственный ключ
    [LocalizableField("custom_quest_start_dialog")]
    public string startDialog = "Are you ready for an adventure?";

    // И даже с вложенными классами!
    [System.Serializable]
    public class ItemInfo
    {
        [LocalizableField] public string Name;
        [LocalizableField] public string Description;
    }

    [LocalizableField]
    private ItemInfo magicSword;
}
