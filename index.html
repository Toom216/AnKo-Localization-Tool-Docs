Документация по инструменту локализации для Unity

Этот инструмент представляет собой комплексное решение для локализации игр и приложений в Unity. Он позволяет автоматизировать процесс сбора текста и ассетов, управлять переводами через мощный интерфейс, интегрироваться с сервисами машинного перевода и динамически обновлять локализованный контент в игре.

👑 Ключевые возможности

Core Features & Workflow

  • Мощный редактор таблицы переводов: Полноценный интерфейс для управления всеми ключами и переводами прямо в Unity с полной поддержкой Undo/Redo, автосохранением и восстановлением сессии.
  • Умная группировка ключей: Автоматически объединяет ключи для множественного числа и рода (например, apple_one, apple_few) в сворачиваемые группы.
  • Продвинутая система парсинга: Автоматический поиск текста в сценах, префабах, UI Toolkit (UXML) и C# скриптах (включая поля с атрибутом [LocalizableField]).
  • Локализация любых ассетов: Поддержка спрайтов, текстур, аудиоклипов, материалов, видео, префабов и других типов ассетов.
  • Live Updates (Живые обновления): Загрузка актуальных переводов с удаленного URL (Google Sheets или JSON) при старте игры с кэшированием.

Automation & Tools

  • Контекстный поиск по проекту: Асинхронный инструмент "Find" для мгновенного нахождения любого локализованного элемента.
  • Детальный отчет о парсинге: На вкладке 'Report' доступен подробный отчет со всеми изменениями (Added, Updated, Removed и т.д.).
  • Импорт и экспорт данных: Поддержка форматов CSV, XML, YAML и XLIFF, а также прямой импорт из Google Sheets.

🚀 Быстрый старт и настройка

1. Установка

Скопируйте ассет в папку вашего проекта. Инструмент автоматически предложит установить необходимые зависимости. После установки откройте окно через Tools -> Localization Tool.

2. Первоначальная настройка

  1. Настройка языков: На вкладке Settings -> Language Management выберите ваш основной язык как Source Language и включите (Enabled) все целевые языки.
  2. Указание контента: На вкладке Content добавьте ваши сцены в Scenes to Parse и папки с префабами в Prefab Folders.
  3. Первый запуск: Перейдите на вкладку Actions и нажмите Update Keys. Инструмент просканирует проект и автоматически добавит необходимые компоненты.

⚙️ Основные компоненты

Важно: Все компоненты добавляются на объекты автоматически при нажатии Update Keys. Вам не нужно делать это вручную.

  • LocalizedText: Основной компонент для текста (Text, TMP_Text).
  • LocalizedAsset: Для подмены ассетов (Sprite, AudioClip, Material). Корректно работает с Play on Awake.
  • LocalizedPrefab: Для неразрушающей локализации префабов.
  • UITKLocalization: Управляет локализацией внутри UI Toolkit (UXML) документов.
  • LocalizedBehaviour: Позволяет вашим скриптам реагировать на смену языка через атрибут [OnLanguageChange].

🛠️ Окно "Localization Tool"

Это ваш главный центр управления. Здесь есть несколько вкладок:

  • Settings: Глобальные настройки, выбор режима генерации ключей, управление языками.
  • Content: Указание сцен, префабов и динамического текста для парсинга.
  • Actions: Запуск парсера, импорт/экспорт, авто-перевод.
  • Report: Подробный отчет после каждого парсинга. Позволяет отследить все изменения и найти любой ключ в проекте.

📝 Редактор переводов ("Translation Table Editor")

Отдельное окно (Tools -> Localization -> Translation Table Editor) для ручного редактирования переводов.

  • Умная группировка ключей для множественного числа и рода.
  • Продвинутое редактирование с окном MultiLineEditWindow, которое показывает исходный текст и валидирует плейсхолдеры в реальном времени.
  • Полная поддержка Undo/Redo (Ctrl+Z / Ctrl+Y).
  • Встроенный менеджер бэкапов и автосохранение сессии.

💡 Примеры использования

Атрибут [LocalizableField]

Предпочтительный способ для локализации полей в инспекторе. Работает со строками, списками и даже вложенными классами.

using Ankonoanko.Localization;
using UnityEngine;

public class QuestManager : MonoBehaviour
{
    // Парсер найдет это поле
    [LocalizableField]
    private string defaultQuestFailedMessage = "You have failed the quest.";

    // Можно задать собственный ключ
    [LocalizableField("custom_quest_start_dialog")]
    public string startDialog = "Are you ready for an adventure?";
}

Функция _() и атрибут [OnLanguageChange]

Используйте функцию _() для динамического текста, который меняется в ходе игры.

using static Ankonoanko.Localization.LocalizationManager;
using Ankonoanko.Localization;
using UnityEngine;

public class PlayerHUD : MonoBehaviour
{
    [OnLanguageChange]
    void UpdateUI()
    {
        int score = 100;
        // Ключ: "score_label", Текст: "Score: {0}"
        scoreText.text = _("score_label", score); 
    }
}

⚠️ Важные нюансы

  • Резервные копии: Всегда создавайте бэкап перед глобальными операциями.
  • Папка StreamingAssets: Файлы переводов должны находиться в Assets/StreamingAssets/, чтобы попасть в билд игры.
  • Исключение LanguageSelector: Объект с компонентом LanguageSelector необходимо добавить в список игнорирования на вкладке Content.
