1. 🚀 Введение

Этот инструмент представляет собой комплексное решение для локализации игр и приложений в Unity. Он позволяет автоматизировать процесс сбора текста и ассетов, управлять переводами через мощный интерфейс, интегрироваться с сервисами машинного перевода и динамически обновлять локализованный контент в игре.

Ключевые возможности

  • Продвинутый парсинг: Автоматическое сканирование сцен, префабов, UI Toolkit (UXML) и C# скриптов, включая поля, отмеченные атрибутом [LocalizableField], а также вложенные классы и списки.
  • Локализация любых ассетов: Управление спрайтами, аудио, префабами, материалами и т.д.
  • Мощный редактор переводов: Централизованный интерфейс с автосохранением, полной поддержкой Undo/Redo и умной группировкой ключей.
  • Машинный перевод: Интеграция с DeepL, Google Translate и Microsoft Translator с гибкими настройками (размер пакета, количество повторных попыток) для каждого сервиса.
  • Гибкий импорт/экспорт: Поддержка CSV, XML, YAML, XLIFF и прямой импорт из Google Sheets.
  • Live Preview в редакторе: Предпросмотр любого языка, не запуская игру.
  • Поддержка Plural & Gender: Корректная обработка форм множественного числа (с правилами для славянских, арабского и других языков) и рода.
  • Полная поддержка RTL: Корректное отображение языков с написанием справа налево.
  • Live Updates: Загрузка актуальных переводов с удаленного сервера при старте игры.
  • Менеджер бэкапов: Встроенный инструмент для создания и восстановления резервных копий.

2. 🛠️ Быстрый старт и настройка

2.1. Установка

  1. Скопируйте ассет в папку вашего проекта Unity.
  2. Инструмент автоматически проверит наличие необходимых зависимостей. В появившемся диалоговом окне установщика подтвердите установку.
  3. Обязательные зависимости (Newtonsoft Json, Editor Coroutines) необходимы для базовой работы.
  4. Опциональные зависимости (Arabic Support, CsvHelper, YamlDotNet) включают дополнительные функции.
  5. Нажмите Install Selected, чтобы установить рекомендуемые пакеты.
  6. После установки откройте главное окно инструмента через меню Tools -> Localization Tool.
[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот окна установщика зависимостей ---]

2.2. Первоначальная настройка

  1. Создание настроек: При первом открытии инструмент создаст файл настроек LocalizationSettings.asset в папке Assets/Resources.
  2. Настройка языков: На вкладке Settings в секции Language Management убедитесь, что ваш основной язык (например, en — английский) выбран как Source Language. Включите (Enabled) все языки, которые вы планируете поддерживать.
  3. Указание контента для парсинга: На вкладке Content добавьте все сцены для анализа в список Scenes to Parse. Если вы используете префабы с текстом, убедитесь, что папки с ними добавлены в Prefab Folders.
  4. Первый запуск парсера: Перейдите на вкладку Actions и нажмите кнопку Update Keys. Инструмент просканирует ваш проект, создаст файлы с переводами и автоматически добавит необходимые компоненты (LocalizedText, LocalizedAsset и т.д.) на игровые объекты.

3. 🧩 Основные компоненты (назначаются автоматически)

Важное замечание: Все описанные ниже компоненты добавляются на игровые объекты автоматически в процессе парсинга (когда вы нажимаете кнопку Update Keys). Вам не нужно добавлять их вручную.

Эти компоненты — "мост" между вашими объектами в сцене и базой данных переводов. Они "слушают" смену языка и автоматически подставляют нужный текст или ассет.

3.1. LocalizedText

Основной компонент для отображения переведенного текста. Устанавливается на объекты с Text, TMP_Text и TextMesh.

  • localizationKey: Ключ, по которому находится перевод. Генерируется автоматически.
  • isStyleOnly: Если true, компонент будет применять только стили (шрифт, RTL), но не менять сам текст. Полезно для элементов, текст которых управляется другим скриптом (например, LanguageSelector).
  • originalSourceText: Исходный текст на базовом языке. Используется как резервный вариант.

3.2. LocalizedAsset

Используется для подмены ассетов (Sprite, AudioClip, Material и т.д.). Он автоматически определяет тип целевого компонента на объекте (Image, AudioSource) и подменяет его ресурс.

Для компонентов с опцией Play on Awake (таких как AudioSource, VideoPlayer), LocalizedAsset корректно перехватывает автозапуск, подменяет ассет и затем запускает воспроизведение, чтобы избежать проигрывания нелокализованного контента.

3.3. LocalizedPrefab

Компонент для локализации целых префабов. Он работает неразрушающим образом: не изменяет исходный префаб, а создает экземпляр локализованной версии как дочерний объект, отключая все скрипты (MonoBehaviour), рендереры (Renderer) и коллайдеры (Collider) на оригинальном объекте. Это предотвращает двойное выполнение логики и появление визуальных артефактов. Для корректной работы в рантайме его порядок выполнения установлен на -100 ([DefaultExecutionOrder(-100)]), чтобы он срабатывал раньше других скриптов.

3.4. UITKLocalization

Добавляется на объекты с UIDocument и управляет локализацией всех текстовых элементов внутри UI Toolkit документа (UXML).

3.5. LocalizedDropdown

Добавляется на Dropdown и TMP_Dropdown для перевода их вариантов.

3.6. LocalizedBehaviour

Служебный компонент, который позволяет вашим скриптам реагировать на смену языка. Он автоматически находит и вызывает методы, помеченные атрибутом [OnLanguageChange].

3.7. Контекстное меню "Analyze for Localization"

Для быстрого добавления LocalizedAsset на объект, вы можете кликнуть правой кнопкой мыши по нужному компоненту (например, Image, AudioSource) в инспекторе и выбрать Analyze for Localization. Инструмент сам добавит и настроит компонент. (Это необязательно, но можно использовать для мануального добавления).

4. 🖼️ Окно "Localization Tool"

4.1. Предпросмотр в редакторе (In-Editor Preview)

Прямо под заголовком находится выпадающий список Preview Language. Эта мощная функция позволяет увидеть, как будет выглядеть локализация на любом языке прямо в окне Scene, не запуская игру.

[--- МЕСТО ДЛЯ GIF: Демонстрация работы Live Preview в редакторе Unity ---]
  • Как это работает: Выберите язык из списка, и инструмент мгновенно применит соответствующие переводы, шрифты, RTL-настройки и ассеты ко всем локализуемым объектам на активной сцене.
  • Плейсхолдеры: Если на вкладке Settings выбран стиль плейсхолдеров (например, Accents или Brackets), то в режиме превью вместо реальных переводов будут отображаться эти плейсхолдеры. Это идеально подходит для тестирования верстки и поиска нелокализованных элементов.
  • Безопасность: Все изменения, внесенные в режиме превью, являются временными. Инструмент автоматически вернет все в исходное состояние при выборе "Revert to Original", закрытии окна, смене сцены или перед сохранением сцены/префаба (благодаря компоненту LocalizationPreviewProtector).
  • Защита от сбоев: Встроенная система PreviewCrashProtector автоматически восстановит исходное состояние объектов, если редактор Unity закроется аварийно во время активного превью.

4.2. Вкладка "Settings" (Настройки)

Ваш центр управления. Здесь вы определяете глобальные правила для всего процесса локализации.

[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот вкладки "Settings" ---]
  • Key Generation Mode: Выберите, как будут создаваться ключи.
    • UseTextAsKey: Ключом становится сам текст. Идеально для прототипов. Минус: если изменить исходный текст, ключ изменится, и все его переводы будут потеряны.
    • AutoGenerateKeysOnly: Ключ генерируется на основе иерархии и имени объекта. Надежно для продакшена. Плюс: переводы не ломаются при изменении текста.
    • UseTextAsKeyWithCustomPriority и AutoGenerateWithCustomKeys: Гибридные режимы, позволяющие задавать кастомные ключи в коде через атрибут [LocalizableField("my_custom_key")].
    • Безопасная миграция: Вы можете сменить режим в любой момент. Инструмент автоматически перенесет все существующие переводы на новую систему ключей.
  • Language Management: Настройте список языков. Для языков с особыми символами назначьте соответствующий Font Asset. Включите опцию RTL для языков с письмом справа налево.
  • General Settings:
    • Parse Prefabs: Включает парсинг префабов.
    • Split files by language: Определяет, как хранить переводы (один большой файл или по файлу на язык).
    • Translations Path: Путь для хранения файлов .json с переводами. Важно: папка должна находиться внутри Assets/StreamingAssets/.
  • Debugging & Testing:
    • Placeholder Style: Выберите стиль для отображения плейсхолдеров в режиме превью.
  • Live Updates: Настройки для загрузки переводов с удаленного сервера.
  • Runtime API Key: Секция для безопасного хранения API ключа, который может понадобиться в скомпилированной игре (хранится в зашифрованном виде).

4.3. Вкладка "Content" (Контент)

Здесь вы сообщаете инструменту, где именно искать текст.

[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот вкладки "Content" ---]
  • Scenes to Parse: Перетащите сюда все сцены для анализа.
  • Prefab Folders: Укажите папки с префабами.
  • Dynamic Texts: Впишите сюда строки, которые создаются исключительно в коде (например, "Game Over").
  • Parsing Ignores: Укажите скрипты, компоненты или объекты для игнорирования.
  • Pin: Эта функция позволяет "закрепить" объект из сцены в списке игнорирования. Вместо временной ссылки на объект, инструмент сохранит его полный путь в иерархии, делая игнорирование постоянным между сессиями (но учтите, что переименование объекта или его родителя сломает эту связь).

4.4. Вкладка "Actions" (Действия)

Главная рабочая вкладка.

[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот вкладки "Actions" ---]
  • Update Keys: Запускает парсер, который обновляет ваши файлы переводов.
  • Open Translation Editor: Открывает отдельное, более удобное окно для редактирования всех переводов.
  • Data Management: Используйте для обмена данными с переводчиками (Export/Import в CSV/XML/YAML/XLIFF, импорт из Google Sheets).
  • Auto-Translation: Автоматически заполняет все пустые строки переводов. Новые настройки позволяют управлять размером пакета (Batch Size) и политикой повторных попыток (Retry Policy) для каждого сервиса (DeepL, Google, Microsoft) отдельно. Инструмент отслеживает количество переведенных символов и предупреждает о возможном превышении лимита.
  • Danger Zone: Содержит кнопки для полного удаления всех компонентов локализации из проекта. Используйте с осторожностью!

4.5. Вкладка "Assets" (Ассеты)

Эта вкладка полностью посвящена локализации нетекстовых ресурсов.

[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот вкладки "Assets" ---]
  1. Создайте структуру папок (опционально): В секции 2. Asset Folder Generation нажмите Create Asset Folders Now.
  2. Настройте категории и правила именования: В секции 3. Asset Categories & Scanning убедитесь, что правило Naming Rule ({key}_{lang}) соответствует вашим файлам (например, button_ok_en.png).
  3. Разместите ваши ассеты: Положите локализованные ассеты в папки.
  4. Просканируйте ассеты: В секции 4. Automation нажмите Scan Assets & Update Tables. Процесс очистки стал безопаснее: теперь удаляются только старые файлы таблиц (.asset), а не вся папка.
  5. Привяжите ассеты к объектам: Нажмите Analyze Project & Attach Components.

4.6. Вкладка "Report" (Отчет)

После каждого парсинга этот отчет показывает полную картину состояния вашей локализации.

[--- МЕСТО ДЛЯ ИЗОБРАЖЕНИЯ: Скриншот вкладки "Report" с примерами записей ---]
  • Категории: All Keys (все ключи), Added (новые), Updated (текст изменился), Removed (удаленные), Duplicates (дубликаты), Migrated (перенесенные), Skipped (неизменные).
  • Продвинутый поиск: Нажмите кнопку Find напротив любой записи, чтобы мгновенно найти соответствующий объект в проекте. Поиск работает асинхронно, не блокируя редактор, и ищет по всем сценам и префабам. Если ключ используется в нескольких местах, появится выпадающий список со всеми источниками.

5. 📝 Редактор переводов ("Translation Table Editor")

Открывается через Tools -> Localization -> Translation Table Editor. Это основной инструмент для ручного редактирования переводов.

[--- МЕСТО ДЛЯ GIF: Работа в редакторе переводов - группировка, редактирование, Undo/Redo ---]
  • Умная группировка: Ключи для множественного числа и рода (например, apple_count_one, apple_count_few) автоматически объединяются в сворачиваемые группы.
  • Продвинутое редактирование: Кликните на ячейку, чтобы открыть всплывающее окно MultiLineEditWindow. Оно показывает исходный текст для сравнения, позволяет быстро его скопировать и в реальном времени проверяет совпадение плейсхолдеров (например, {username}), подсвечивая недостающие или лишние.
  • Валидация плейсхолдеров через комментарии: Вы можете явно указать, какие плейсхолдеры должны быть в переводе, добавив в комментарий к ключу специальную директиву, например: @placeholders: {username}, {score}. MultiLineEditWindow будет использовать этот список как основной источник правды.
  • Навигация и управление: Перемещайтесь по таблице скроллбарами или зажав среднюю кнопку мыши. Изменяйте ширину колонок, перетаскивая разделители. Сохраняйте изменения по Ctrl+S.
  • Полный Undo/Redo: Весь стек действий (изменение текста, добавление/удаление ключей) полностью поддерживается через Ctrl+Z / Ctrl+Y.
  • Резервные копии и автосохранение: Используйте кнопку Manage Backups. Инструмент также автоматически сохраняет вашу сессию каждые несколько минут. При аварийном закрытии будет предложено восстановить изменения.

6. 💡 Примеры использования

6.1. Готовые компоненты и примеры

Проект содержит готовые к использованию скрипты, которые служат отличными примерами.

  • LanguageSelector.cs: Готовый компонент для создания UI выпадающего списка для смены языка. Он автоматически находит все доступные языки и управляет их переключением.
    Как использовать: Просто добавьте компонент LanguageSelector на вашу сцену (например, на пустой GameObject) и укажите в инспекторе ваш TMP_Dropdown.
  • Примеры в коде: Для изучения продвинутых техник, таких как работа с [LocalizableField], функцией _(), множественным числом и родом, изучите файлы StatPurchaseTest.cs и TestLocalization.cs. Они наглядно демонстрируют реализацию всех основных возможностей инструмента в коде.

6.2. Атрибут [LocalizableField]

Предпочтительный способ для текстов, которые являются частью конфигурации компонента в инспекторе. Парсер теперь умеет работать со строками, списками/массивами строк и даже полями внутри вложенных [System.Serializable] классов.

using Ankonoanko.Localization; // Required for [LocalizableField]
using UnityEngine;
using System.Collections.Generic;

public class QuestManager : MonoBehaviour
{
    // Парсер найдет это поле и создаст для него ключ
    [LocalizableField]
    private string defaultQuestFailedMessage = "You have failed the quest.";
    
    // Можно задать собственный ключ
    [LocalizableField("custom_quest_start_dialog")]
    public string startDialog = "Are you ready for an adventure?";
    
    // Работает со списками
    [LocalizableField]
    private List<string> missionObjectives = new List<string> { "Find the treasure" };
    
    // И даже с вложенными классами!
    [System.Serializable]
    public class ItemInfo
    {
        [LocalizableField] public string Name;
        [LocalizableField] public string Description;
    }
    
    [LocalizableField]
    private ItemInfo magicSword;
}

6.3. Функция _() и атрибут [OnLanguageChange]

Используйте функцию _() для динамического текста, который меняется в ходе игры. Метод, обновляющий UI, следует пометить атрибутом [OnLanguageChange], чтобы он вызывался автоматически при смене языка. Для этого на объект будет автоматически добавлен компонент LocalizedBehaviour.

// Чтобы использовать короткий вызов _(), добавьте эти строки
using static Ankonoanko.Localization.LocalizationManager;
using Ankonoanko.Localization; // Required for [OnLanguageChange]
using UnityEngine;
using TMPro;
using System.Collections.Generic;

public class PlayerHUD : MonoBehaviour
{
    [SerializeField] private TMP_Text scoreText;
    private int score = 100;
    private string playerName = "Alex";
    
    void Start()
    {
        UpdateUI(); // Первоначальное обновление
    }
    
    // Этот метод будет вызван автоматически при смене языка
    [OnLanguageChange]
    void UpdateUI()
    {
        // 1. Индексные плейсхолдеры (как в string.Format)
        // Ключ: "score_label", Текст в файле: "Score: {0}"
        scoreText.text = _("score_label", score); // Результат: "Score: 100"
        
        // 2. Именованные плейсхолдеры (рекомендуется для читаемости)
        // Ключ: "welcome_message", Текст: "Welcome, {username}!"
        string welcomeText = _("welcome_message", new { username = playerName });
        
        // Для сложных случаев или высокой производительности
        var args = new Dictionary<string, object> { { "username", playerName } };
        string welcomeText2 = _("welcome_message", args);
    }
}

6.4. Множественное число и род

Инструмент теперь использует более точные правила для разных языковых групп.

using static Ankonoanko.Localization.LocalizationManager;
using Ankonoanko.Localization;

7. ⚠️ Важные нюансы и предупреждения

  • Резервные копии: Перед глобальными операциями (Update Keys, импорт данных) всегда создавайте резервную копию через Translation Table Editor -> Manage Backups.
  • Безопасность API ключей: Ключ для работы в редакторе хранится локально на вашей машине (в EditorPrefs) и не попадает в репозиторий. Для ключей, которые нужны в билде игры, используйте секцию Runtime API Key.
  • Папка StreamingAssets: Файлы переводов должны находиться в подпапке Assets/StreamingAssets/, чтобы они были включены в сборку игры.
  • "Защита от дурака": Инструмент автоматически защищает вас от случайного сохранения временных данных из режима превью в сцену или префаб.
  • Производительность: Вызов _() является быстрым, но в циклах, которые выполняются каждый кадр (например, в Update), старайтесь кэшировать результат в переменную.
  • Исключение LanguageSelector из парсинга: Объект с компонентом LanguageSelector и его TMP_Dropdown необходимо добавить в список Ignore Specific Objects на вкладке Content. Это нужно, чтобы парсер не создавал лишние ключи для вариантов в дропдауне, так как скрипт LanguageSelector заполняет их динамически во время выполнения. Игнорирование предотвращает конфликты и сохраняет чистоту в файлах переводов.
  • Шрифты для LanguageSelector: Чтобы выбранный язык в TMP_Dropdown корректно обновлял свой шрифт, вручную добавьте на дочерний объект Label у Dropdown пустой компонент LocalizedText и включите у него галочку isStyleOnly.

8. 🧬 Расширение функционала

8.1. Создание кастомного парсера

Вы можете легко расширить систему для поддержки кастомных компонентов, создав свой класс-парсер.

  1. Создайте новый C# скрипт в папке Editor вашего проекта.
  2. Скопируйте в него приведенный ниже шаблон.
  3. Измените логику, чтобы она соответствовала вашему компоненту.
#if UNITY_EDITOR
using System.Collections.Generic;
using UnityEngine;
using Ankonoanko.Localization;

// ШАБЛОН для создания парсера для кастомного компонента.
public class CustomComponentParser_Template : ITextComponentParser
{
    public IEnumerable<(string text, string key, string source)> Parse(
        GameObject gameObject,
        LocalizationSettings settings)
    {
        // 1. Проверяем, есть ли на объекте нужный нам кастомный компонент.
        var component = gameObject.GetComponent<AwesomeComponentFromAssetStore>();
        if (component == null)
        {
            yield break; // Выходим, если компонента нет
        }
        
        // 2. Получаем текст из полей кастомного компонента.
        string titleText = component.Title;
        
        // 3. Проверяем, является ли текст валидным для локализации.
        if (TextParser.IsValidLocalizableText(titleText))
        {
            // 4. Генерируем ключ и источник, используя хелперы из TextParser.
            string key = TextParser.GenerateKeyForObject(gameObject, titleText, settings.keyGenerationMode);
            string source = TextParser.GetSourceStringForObject(gameObject);
            
            // 5. Возвращаем результат.
            yield return (titleText, key, source);
        }
        
        // 6. Повторяем для других полей...
        // (можно добавить суффикс, чтобы ключи были уникальными)
        string descriptionText = component.Description;
        if (TextParser.IsValidLocalizableText(descriptionText))
        {
            string key = TextParser.GenerateKeyForObject(gameObject, descriptionText, settings.keyGenerationMode) + "_description";
            string source = TextParser.GetSourceStringForObject(gameObject);
            yield return (descriptionText, key, source);
        }
    }
}
#endif
